    29  // GetSdkHeaders - returns the set of SDK-specific headers to be included in an outgoing request.
    30  //
    31  // This function is invoked by generated service methods (i.e. methods which implement the REST API operations
    32  // defined within the API definition). The purpose of this function is to give the SDK implementor the opportunity
    33  // to provide SDK-specific HTTP headers that will be sent with an outgoing REST API request.
    34  // This function is invoked for each invocation of a generated service method,
    35  // so the set of HTTP headers could be request-specific.
    36  // As an optimization, if your SDK will be returning the same set of HTTP headers for each invocation of this
    37  // function, it is recommended that you initialize the returned map just once (perhaps by using
    38  // lazy initialization) and simply return it each time the function is invoked, instead of building it each time
    39  // as in the example below.
    40  //
    41  // If you plan to gather metrics for your SDK, the User-Agent header value must
    42  // be a string similar to the following:
    43  // my-go-sdk/0.0.1 (lang=go; arch=x86_64; os=Linux; go.version=1.12.9)
    44  //
    45  // In the example above, the analytics tool will parse the user-agent header and
    46  // use the following properties:
    47  // "my-go-sdk" - the name of your sdk
    48  // "0.0.1"- the version of your sdk
    49  // "lang=go" - the language of the current sdk
    50  // "arch=x86_64; os=Linux; go.version=1.12.9" - system information
    51  //
    52  // Note: It is very important that the sdk name ends with the string `-sdk`,
    53  // as the analytics data collector uses this to gather usage data.
    54  //
    55  // Parameters:
    56  //
    57  //	serviceName - the name of the service as defined in the API definition (e.g. "MyService1")
    58  //	serviceVersion - the version of the service as defined in the API definition (e.g. "V1")
    59  //	operationId - the operationId as defined in the API definition (e.g. getContext)
    60  //
    61  // Returns:
    62  //
    63  //	a Map which contains the set of headers to be included in the REST API request
